# Contributing to Living Architecture

Thank you for your interest in this project.

## How to Engage

This repository is an open mathematical and architectural draft. It welcomes:

- **Mathematical analysis** — rigorous examination of the formulations
- **Implementation work** — translating the architecture into working systems
- **Testing** — empirical validation of the proposed mechanisms
- **Constructive criticism** — identifying gaps, inconsistencies, or improvements

## Types of Contributions

Please distinguish among:

| Type | Description |
|------|-------------|
| **Mathematical inconsistency** | A demonstrated logical or mathematical error in the formulations |
| **Implementation limitation** | A practical constraint that does not invalidate the theory |
| **Architectural alternative** | A different approach to the same problem |
| **Editorial correction** | Typos, formatting, or clarity improvements |

## Proposing Mathematical Corrections

If you believe a formula contains an error, please identify:

1. The specific formula being questioned
2. The exact inconsistency or contradiction
3. The assumptions under which the inconsistency occurs
4. A proposed replacement and its consequences for the rest of the architecture

> **Note:** A richer representation in a later version does not automatically prove that an earlier formula was wrong. Version 3 adds to Version 2 — it does not replace it.

## Submitting Changes

1. Fork this repository
2. Create a branch with a descriptive name
3. Make your changes with clear commit messages
4. Open a Pull Request with a detailed description

## Code of Conduct

Be respectful, constructive, and specific. This project explores ideas at the boundary of current knowledge — disagreement is welcome, dismissiveness is not.
